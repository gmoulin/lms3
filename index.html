LMS stands for Library Management System

For the third version I wanted to test a full javascript environment using Nodejs, Angularjs and Sass. As the same time I wanted this project to be part of a public portfolio.

See problems and solutions found along the way.

environment aware configuration

node, grunt / gulp

clear folder hierarchy

It is really important to have a clear organisation for files and folder to make development and maintenance easier.

A client folder for angular files, sass stylesheets, fontend tests files, ...
A server folder for node files, backend tests files, ...
A public folder for images, compiled templates, scripts and stylesheets, filed mainly by grunt / gulp.
A bower_components and a node_modules folders for third party components.

The public folder will be the publication folder for the 3 environments.
